# TIR → RGB: Thermal Super-Resolution + Colorization (BAH 2026 · PS10 · SAC-ISRO)

End-to-end trained deep-learning pipeline for **Bharatiya Antariksh Hackathon 2026, PS10**:

```
Input:    single-channel Thermal IR tile @ 200 m  (256×256)
Output 1: super-resolved TIR @ 100 m              (512×512)   — fine-tuned ×2 RRDBNet
Output 2: colorized RGB @ 100 m                   (512×512×3) — Pix2Pix (U-Net + PatchGAN)
```

## Architecture

| Stage | Model | Training |
|---|---|---|
| Super-resolution ×2 | RRDBNet (Real-ESRGAN arch), **fine-tuned on TIR** from RealESRGAN_x2plus weights | L1-dominant loss (hallucination-averse), paired 200 m→100 m patches |
| Colorization | Pix2Pix, U-Net256 generator + PatchGAN discriminator | 100 m TIR → 100 m RGB paired tiles, L1 λ=100 + cGAN loss |

**Physics-informed design:** all thermal data is normalized against a *fixed*
brightness-temperature window (270–330 K) instead of per-patch stretching, so pixel
values keep a stable physical meaning (temperature) across the entire dataset and at
inference time. RGB targets use fixed surface-reflectance scaling. See
`src/data/normalization.py`.

## Data

Landsat-9 Collection-2 Level-2 (`LC09/C02/T1_L2`), fetched via Google Earth Engine for
~30 Indian regions × N×N adjacent tiles (coastal, arid, mountain, urban, agricultural).
Each 51.2 km region yields co-registered `tir_200m` (256²), `tir_100m` (512²), and
`rgb_100m` (512²) arrays — resampling done server-side by GEE, so pairs are aligned by
construction. Train/val split is **by city**, so validation is unseen geography.

## Quickstart (Colab)

Open `notebooks/colab_driver.ipynb` and run top to bottom. Stages:

```bash
# 1. Download raw tiles (needs a GEE account/project)
python -m src.data.download_gee --project <gee-project> --out data/raw --grid 3

# 2. Build training sets for both stages (+ alignment sanity gallery)
python -m src.data.prepare_patches --raw data/raw --out data

# 3a. Train colorization (Kaggle/Colab GPU, ~150+50 epochs)
python -m src.train.train_pix2pix --data data/tir2rgb --ckpt checkpoints --name tir2rgb

# 3b. Fine-tune SR (~20k iters)
wget -P weights https://github.com/xinntao/Real-ESRGAN/releases/download/v0.2.1/RealESRGAN_x2plus.pth
python -m src.train.train_sr --data data/sr --pretrained weights/RealESRGAN_x2plus.pth --out checkpoints/sr

# 4. Evaluate (PSNR/SSIM/FID + water-consistency + timing + gallery)
python -m src.eval.metrics --raw data/raw --sr-ckpt checkpoints/sr/sr_best.pth \
    --p2p-ckpt checkpoints/tir2rgb/latest_net_G.pth --out results/eval --fid

# 5. Single-tile inference
python -m src.infer.pipeline --tir data/raw/Pune_11.npz \
    --sr-ckpt checkpoints/sr/sr_best.pth --p2p-ckpt checkpoints/tir2rgb/latest_net_G.pth

# 6. Demo app
python -m app.demo_gradio --sr-ckpt checkpoints/sr/sr_best.pth \
    --p2p-ckpt checkpoints/tir2rgb/latest_net_G.pth --share
```

## Evaluation-criteria mapping

- **PSNR / SSIM / FID** — `src/eval/metrics.py`, reported per stage.
- **Hallucination check** — L1-dominant SR loss + water-consistency check in eval and
  live in the demo log; visual gallery per val tile in `results/eval/gallery/`.
- **Inference time** — measured per stage per tile, shown in demo UI.
- **Physics-informed (bonus)** — fixed radiometric normalization (see above).
